# 系统层次校验

在开始分析之前，确认当前系统的层次定位。

## 层次判断

| 层次 | 系统定义 | 目标的来源 | 此系统是否匹配？ |
|------|---------|----------|---------------|
| 物理系统 | 由自然定律自封闭 | 内部物理/技术指标 | |
| 工程系统 | 物理系统 + 制造/装配可行性 | 性能-可制造性权衡 | |
| 产品系统 | 工程系统 + 使用场景 + 生命周期 | 使用场景赋予的外部目的 | |
| 社会-技术系统 | 产品系统 + 组织/市场/制度 | 长期存续与演化能力 | |

## 关键检查

- 如果系统有明确的终端用户 → 它是产品系统，目标不由内部指标定义
- 如果系统是某个更大系统的子组件 → 目标必须由母系统向下分解
- 仅在系统完全自封闭时 → 才可以用内部物理/技术指标作为目标

## 层错预防

- 当前定义的「优化目标」是否在正确的层次上？
- 是否存在「在物理层优化产品层问题」的层次错位风险？
